# 登录系统第二次改进 — 新增注册与搜索功能

| 维度 | 改进前（第一次作业） | 改进后（第二次作业） |
|------|-------------------|-------------------|
| **数据存储** | 使用 `data/users.json` 文件存储用户数据 | 升级为 SQLite 数据库 `data/users.db`，支持更高效的数据管理 |
| **用户注册** | 无注册功能，仅能使用预设账号登录 | 新增 `/register` 注册页面，用户可自行注册账号 |
| **用户搜索** | 无搜索功能 | 新增搜索功能，支持按用户名和邮箱模糊搜索用户 |
| **数据库操作安全** | 使用参数化查询，安全性较高 | **故意使用 f-string SQL 拼接**，用于 SQL 注入教学演示 |
| **前端页面** | 仅首页和登录页，功能单一 | 新增 `register.html` 注册页面，首页新增搜索面板和结果表格 |
| **导航栏** | 未登录时仅显示"登录"链接 | 未登录时新增"注册"链接，方便用户快速注册 |
| **登录页** | 登录错误提示功能 | 保留错误提示，新增成功提示（注册成功后显示"注册成功，请登录"） |
| **默认用户** | admin/admin123、alice/alice2025（JSON 存储） | 保留相同默认用户，通过 SQLite INSERT OR IGNORE 自动初始化 |
| **会话安全** | 保留第一次作业的会话加固配置 | 完全保留：HttpOnly + SameSite + 30 分钟超时 + 随机密钥 |

## 新增路由说明

### `/register`（GET/POST）
- GET：显示注册表单（用户名、密码、邮箱、手机号）
- POST：使用 f-string 拼接 SQL 插入用户数据到 SQLite，注册成功后跳转到登录页
- 控制台打印执行的 SQL 语句

### `/search` → `/?keyword=xxx`（GET）
- 在首页嵌入搜索框，通过 URL 参数 `keyword` 传递关键词
- SQL 使用 f-string 拼接：`LIKE '%{keyword}%'`，控制台打印完整 SQL
- 搜索结果以表格展示：ID、用户名、邮箱、手机

## 安全说明（教学用途）

> ⚠️ 注册和搜索功能的 SQL 查询**故意使用 f-string 字符串拼接**，未做任何输入过滤或转义，控制台会打印执行的 SQL 语句。此设计用于 **SQL 注入教学演示**，请勿在生产环境中使用。
>
> 生产环境应使用参数化查询（`?` 占位符），如本程序中的 `get_user_by_username()` 函数所示。

## 改进成果总结

- **功能扩展**：从仅有登录功能扩展到登录 + 注册 + 搜索完整功能链
- **数据升级**：从 JSON 文件存储升级为 SQLite 关系型数据库
- **用户体验**：新增注册入口、搜索面板、结果表格、成功提示等交互元素
- **教学价值**：保留 SQL 注入演示场景，便于学习 Web 安全
- **版本兼容**：保留第一次作业的所有安全加固配置，迭代不退化
