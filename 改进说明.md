# 第四次作业改进说明

## 新增功能

### 1. 个人中心（/profile）
- 新增 `/profile` 路由，支持通过 URL 参数 `user_id` 查询任意用户的资料
- 展示用户 ID、用户名、邮箱、手机号、余额等信息
- **未做权限校验**：不验证当前登录用户和要查询的 user_id 是否匹配，管理员和普通用户的资料都可以通过修改 URL 参数查看
- 新增 `get_user_by_id()` 函数，按用户 ID 查询数据库

### 2. 账户充值（/recharge）
- 新增 `/recharge` 路由，仅支持 POST 方法
- 从表单接收 `user_id` 和 `amount` 参数
- 直接执行 `balance = balance + amount` 更新余额
- **未做金额校验**：不检查 amount 是否为负数
- 充值成功后重定向回 `/profile?user_id={user_id}`

### 3. 数据库架构变更
- 在 `users` 表中新增 `balance` 字段（REAL 类型，默认值 0）
- 兼容已有数据库：通过 `ALTER TABLE` 尝试添加字段，捕获异常避免重复添加
- 预设 admin 余额为 99999，alice 余额为 100

### 4. 新增页面模板
- **profile.html**：个人中心页面，继承 base.html
  - 用户信息展示区（ID、用户名、邮箱、手机、余额）
  - 充值表单（金额输入框 + 充值按钮）
  - 隐藏字段传递 user_id
  - 绿色渐变的充值按钮样式

### 5. 导航栏与首页更新
- **base.html**：登录后的菜单新增"个人中心"链接，指向 `/profile?user_id=1`
- **index.html**：首页欢迎区域新增"个人中心"快捷入口按钮

## 安全说明（已知问题，为教学演示保留）
- `/profile` 未校验访问权限，可查看任意用户资料
- `/recharge` 未校验金额正负，可充负数或任意值
- 搜索功能仍存在 SQL 注入漏洞（保留用于教学演示）
- 注册功能仍存在 SQL 注入漏洞（保留用于教学演示）
