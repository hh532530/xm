# 登录系统第三次改进 — 新增用户头像上传功能

| 维度 | 改进前（第二次作业） | 改进后（第三次作业） |
|------|--------------------|--------------------|
| **用户头像上传** | 无上传功能 | 新增 `/upload` 路由，支持用户上传头像文件 |
| **上传限制** | 无 | 设置 `MAX_CONTENT_LENGTH = 16MB`，限制上传文件大小 |
| **文件存储** | 无 | 文件保存至 `static/uploads/` 目录，使用原始文件名保存 |
| **文件校验** | — | **故意不做任何检查**：不检查后缀名、不检查 MIME 类型，用于文件上传漏洞教学演示 |
| **前端页面** | 首页、登录、注册 3 个页面 | 新增 `upload.html` 上传页面 |
| **导航栏** | 登录后显示"欢迎"和"退出" | 登录后新增"上传头像"链接 |
| **首页** | 仅显示用户信息和搜索功能 | 新增"上传头像"快捷按钮入口 |
| **上传预览** | 无 | 上传成功后显示圆形图片预览和文件访问链接 |

## 新增路由说明

### `/upload`（GET/POST）
- **需要登录**才能访问，未登录自动跳转到登录页
- GET：显示上传表单（文件选择 + 上传按钮）
- POST：接收上传文件，以**原始文件名**保存到 `static/uploads/` 目录
- 不检查文件后缀、不检查 MIME 类型
- 上传成功后显示图片预览和文件访问 URL

## 安全说明（教学用途）

> ⚠️ 上传功能**故意不做任何文件类型检查**（不检查后缀名、不检查 MIME 类型），且使用**用户提供的原始文件名**保存。此设计用于**文件上传漏洞教学演示**，请勿在生产环境中使用。
>
> 生产环境应对上传文件进行：白名单后缀检查、MIME 类型验证、文件内容扫描、重命名存储等安全措施。

## 第三次改进成果总结

- **功能扩展**：从登录 + 注册 + 搜索扩展到支持用户头像上传
- **文件上传**：支持任意文件类型上传，保存原始文件名
- **用户体验**：上传成功后即时预览，导航栏和首页均有上传入口
- **教学价值**：保留文件上传漏洞演示场景，便于学习 Web 安全
- **版本兼容**：完全保留前两次作业的所有功能
